Getting Started

API Reference

The AI-ChatBot API is a multi-tenant REST API for deploying and operating conversational agents. Each agent has its own configuration, session history, analytics, and optional knowledge base.

Base URL: http://your-host:8000

Authentication

All endpoints (except POST /agent) require an agent-specific API key passed as a request header.

Header: X-API-Key: <your-api-key>
The API key is generated once on agent creation. Store it securely — it cannot be recovered.

Quickstart

Create an agent, configure its context, and send the first message in three steps.

1. Create an agent

Request
POST /agent
Content-Type: application/json

{
  "name": "My Agent",
  "owner": "you@company.com"
}
Response
{
  "agent_id": "agt_...",
  "api_key": "sk_live_..."
}

2. Set the agent context

Request
PUT /agent/context
X-API-Key: sk_live_...

{
  "persona": "You are a helpful support agent for Loja X.",
  "tone": "formal",
  "language": "pt-BR"
}

3. Send a message

Request
POST /chat
X-API-Key: sk_live_...

{
  "session_id": "sess_abc123",
  "message": "What is your return policy?"
}
Response
{
  "message_id": "msg_...",
  "content": "Our return policy allows returns within 30 days...",
  "usage": { "input_tokens": 120, "output_tokens": 42, "total_tokens": 162 },
  "response_time_ms": 834
}
Agents

Agent Endpoints

POST /agent Create a new agent

Creates an agent and returns the API key. The key is shown only once.

FieldTypeDescription
namestringDisplay name for the agent
ownerstringOwner identifier (email or ID)
tagslist[string]Optional tags for filtering
PUT /agent/context Update agent context

Each update creates a new versioned snapshot. See the Agent context guide for all fields.

GET /agent/context Get current context

Returns the latest version of the agent context.

GET /agent/context/history List context versions

Returns all versioned snapshots of the context with change history.

GET /agent/metrics Agent metrics

Returns aggregated metrics: session count, token usage, average sentiment, escalation rate.

DELETE /agent Delete agent and all data

Permanently removes the agent, all sessions, contexts, and user data. Irreversible.

Chat

Chat Endpoints

POST /chat Send a message

Processes a user message and returns the AI response. Triggers NLP analysis and incremental persistence on every call.

FieldTypeDescription
session_idstringUnique session identifier (generate client-side)
user_idstring?Optional user identifier for context tracking
messagestringThe user's message text
POST /chat/{session_id}/end End a session

Finalizes the session, generates insights, and persists full history. Updates user context if user_id was provided.

POST /chat/{session_id}/escalate Escalate session

Marks the session as escalated. Can be triggered manually or automatically via escalation triggers.

POST /chat/{session_id}/resolve Resolve session

Marks the session as resolved without ending it.

Data

Data Endpoints

GET /data/chat List sessions

Returns all sessions for the authenticated agent.

GET /data/chat/{session_id} Get session history

Returns the full message history. Falls back to the persistence driver if the session has expired from Redis cache.

GET /data/chat/{session_id}/insights Session insights

Returns AI-generated insights for an ended session: key points, suggested actions, summary.

GET /data/analytics Analytics

Aggregated agent analytics: sessions over time, average sentiment, top topics, escalation rate.

Guides

Agent Context

The agent context defines how the AI behaves. Each PUT /agent/context call creates a new versioned snapshot — the full history is accessible via GET /agent/context/history.

FieldTypeDescription
personastringAgent identity and role. Sent directly as the system prompt opening.
tonestringVoice tone: formal, informal, technical, etc.
languagestringPreferred language for responses (e.g. pt-BR, en-US).
segmentstringService segment for categorisation (e.g. ecommerce, b2b).
behaviorstringDetailed behavioral instructions appended to the system prompt.
fallback_messagestringExact response the AI must return when a restricted topic is raised.
restrictions.topicslist[string]Topics the agent is prohibited from discussing.
knowledge_baseobjectData sources for tool use (premium). See Knowledge base guide.
escalation_triggerobjectBackend-only escalation conditions. Never sent to the AI model.

Note: escalation_trigger is evaluated by the backend after every message. It is never included in the system prompt sent to the AI.

Full context example
{
  "persona": "You are Clara, a support agent for Loja X. Be cordial and objective.",
  "tone": "formal",
  "language": "pt-BR",
  "segment": "ecommerce",
  "behavior": "Answer only questions about orders, shipping and returns.",
  "fallback_message": "I can only help with orders and deliveries.",
  "restrictions": {
    "topics": ["politics", "religion", "competitors"]
  },
  "escalation_trigger": {
    "operator": "OR",
    "conditions": [
      { "type": "sentiment", "threshold": 0.4 },
      { "type": "keyword", "values": ["cancel", "refund", "fraud"] }
    ]
  }
}

Escalation Triggers

Escalation conditions are evaluated by the backend after every message — they are never sent to the AI model.

TypeParameterDescription
sentimentthreshold: floatEscalates if avg sentiment < -threshold
keywordvalues: list[str]Escalates if the last message contains any keyword
message_countvalue: intEscalates after N user messages
topicvalues: list[str]Escalates if detected topic is in the list
time_elapsedvalue: float (seconds)Escalates after N seconds since session start
intentvalue: strEscalates if detected intent matches
Example
{
  "operator": "OR",
  "conditions": [
    { "type": "sentiment", "threshold": 0.4 },
    { "type": "keyword", "values": ["refund", "cancel"] },
    { "type": "message_count", "value": 10 }
  ]
}

Knowledge Base

Knowledge base integration is a premium feature. It allows the AI to query external data sources as tools, routing each question to the appropriate source based on context.

Available options

OptionDescriptionStatus
C — File upload CSV, JSON, PDF or Excel indexed locally. Zero client infrastructure required. In development
A — REST API Agent configured with a base URL and optional auth header. AI calls it as a tool. In development
B — SQL connection Read-only connection string. AI generates SQL, validated through 5 security layers before execution. In development
D — Webhook Natural language question sent to a client webhook; structured data returned. In development

Security layers for SQL (Option B)

#LayerDescription
1Read-only userClient provides a DB user with SELECT-only grants. Orientation script provided.
2Schema registryAI only sees the schema of tables listed in allowed_tables. Full database is never exposed.
3Query validatorRejects non-SELECT statements, blocks information_schema, enforces LIMIT, blocks multi-statement inputs.
4Execution constraints5-second statement timeout, row limit and byte limit applied per query.
5Result sanitizerColumns named password, token, cpf, ssn, secret, email are masked before reaching the AI.
knowledge_base field example (REST API)
{
  "knowledge_base": {
    "type": "api",
    "base_url": "https://api.yourstore.com/v1",
    "auth_header": "Bearer <token>"
  }
}

Storage Options

The persistence driver is set via the STORAGE_TYPE environment variable and can be changed at any time without code modifications.

DriverSTORAGE_TYPEBest for
Local local Development and single-server deployments. Data stored as JSON files under DATA_PATH.
Database Database Production. PostgreSQL via SQLAlchemy. Schema generated by invoke setup.
Webhook Webhook Custom integrations. Every write operation triggers a POST to WEBHOOK_URL.

Persistence strategy

The system uses Redis as a short-lived cache (default TTL: 24h) and the configured driver as cold storage. A persistence snapshot is saved on every message — not only when a session ends. This means abandoned sessions never lose data.

When a session expires from Redis, GET /data/chat/{session_id} automatically falls back to the driver to recover the full message history.

Database schema (PostgreSQL)

TableContents
agentsAgent records and API key hashes
agent_contextsVersioned context snapshots
user_contextsPer-user language, segment and form answers
sessionsSession metadata: tokens, timestamps, flags
session_historyFull message history serialised as JSON
scoresNLP scores per session: sentiment, topics, intent
insightsAI-generated insights produced on session end
